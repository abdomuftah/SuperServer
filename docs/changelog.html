Changelog

What changed in SuperServer.

Release history for the stable Multi-PHP installer, including the fixes discovered during clean Ubuntu 26.04 VM testing.

v3.5.2 — CrowdSec firewall-bouncer recovery

stable
  • Recovered the Ubuntu 26.04 firewall-bouncer post-installation failure.
  • Generated and applied a dedicated local bouncer key.
  • Validated Local API health, active enforcement and dpkg completion.
  • Added service, journal and bouncer-log diagnostics.

v3.5.1 — Fully non-interactive execution

maintenance
  • Removed Composer’s root confirmation after plan approval.
  • Fixed strict Bash unbound-variable errors in FPM and domain helpers.
  • Added non-interactive APT, npm, Composer and needrestart behavior.

v3.5.0 — Resilient FPM and package providers

core
  • Added dynamic PHP-FPM listener discovery for sockets and TCP.
  • Added safe provider fallback chains for major components.
  • Moved PHP settings into conf.d/99-snyt.ini fragments.
  • Corrected CrowdSec AppSec configuration and validation.

v3.4.2 — Keyboard checklists

UX
  • Added arrow-key navigation, Space toggling and Enter confirmation.
  • Kept a number/range fallback for limited terminals.
  • Fixed the hidden PHP runtime-check filename.

v3.4.1 — Wizard-first install

UX
  • Moved all package and repository changes after final approval.
  • Fixed optional-component conditions under strict Bash mode.
  • Added safe extension availability handling.

v3.4.0 — Multi-PHP and CrowdSec

major
  • Added PHP 8.1–8.5, extension profiles and Sury integration.
  • Replaced Fail2ban with CrowdSec and optional AppSec/WAF.
  • Added installation plans, management tools and no-email SSL mode.
